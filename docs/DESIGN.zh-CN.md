# 设计与控制实测结论

中文 · English see [DESIGN.md](DESIGN.md)

本文记录这台 eVTOL 通过 MSFS AICtrl DevKit 到底是怎么被控制的（基于真机实测），以及飞行控制器如何建立在这些事实之上。

## 1. DevKit 接口

DevKit 是本机 REST 服务（`127.0.0.1:5000`）：

- `GET /get` → 返回**每个**暴露变量的 `{name, writable, val}` 列表。
- `PUT /set` `{name, val}` → 设置单个变量。
- `PUT /ap_start` / `PUT /ap_stop` → DevKit 自带的自动驾驶（旋翼定点、定高、定航向）。我们基本弃用它，改直接操纵。

随附 PDF 文档只列了子集；**权威**真相是 `/get` 里每个字段的 `writable` 标志。`dump_fields.py` 会打印它。实测：**76 个字段、41 个可写**，本地 GET/SET 往返 **1–2 ms**。

## 2. 控制映射（本机型实测）

| 输入（SimVar） | 作用 | 说明 |
|---|---|---|
| `GENERAL ENG THROTTLE LEVER POSITION:1` | **前后** | 可写且生效。>中性→前进，<中性→后退。**不改变高度。** 油门约 90 时极限地速 ≈ 5 m/s。 |
| `ELEVATOR POSITION` | **升降（爬升/下降）** | +0.8→爬升 ≈7.5 ft/s，−0.4→下降 ≈3.75 ft/s，−0.85→≈8–9 ft/s。约 **±10 ft/s 每单位**。不改变水平。 |
| `PLANE HEADING / BANK / PITCH` | 姿态 | 可写且**生效**——但每帧写它们来"飞"是作弊，会引起侧滑。只用于起点一次性传送。 |
| `RUDDER / AILERON POSITION` | 偏航 / 横滚 | 可写（方向舵尚未做成转向闭环——见局限）。 |
| `VELOCITY_BODY_X` | （横向） | 飞行中**写入不生效**——指令 −12 回读 ≈ 0。因此侧偏无法用体轴速度修。 |
| collective / cyclic / rotor | — | DevKit **根本未暴露**。 |

悬停中性油门在多次传送之间会漂（**约 33–58**），因为传送后旋翼状态不同——所以控制器每次现读它，并按**绝对值**限幅油门，而不是按相对中性的偏置。

## 3. 几何：为什么"只推油门"就能到坪

在风切变起点，机头航向 **≈ 0.3°**，但停机坪方位 **≈ 20°**——机头在坪的*左侧*约 19.6°。而油门推力矢量在机头**右侧约 19°**。二者抵消，合成地速航迹正好**指向坪**。从起点只推油门，就直奔停机坪，**无需任何航向输入**。（早先"锁航向"的做法与此对抗，才出现约 35% 的向右偏移假象。）

## 4. 控制理念

像人开飞机那样飞：**只用操纵**——油门（前后）、升降舵（升降）、以及（将来）方向舵转向。**不**靠写姿态/位置/体轴速度来移动飞机。唯一的姿态/位置写入是**一次性传送到试次起点**（实验复位，不是控制动作）。

## 5. 前飞巡航 —— `_fwd_fly.py`

闭环定速：由"机头投影位移"求导得到前进地速（符号可靠），用**绝对油门 PI** 跟踪目标（如 4 m/s）。目标速度渐入渐出，起停平滑。效果：平滑加速到定速巡航，高度平到约 ±1ft，航迹笔直。油门按绝对上限（≈96）限幅，避免低中性时"喂不饱"。

## 6. 阶梯进近 —— `_step_land.py` / `_throttle_route_loop`

交替**平飞**段（油门前进、升降舵中立）与**下降**段（升降舵下压）。关键细节：两种段都保持油门**持续**给。若在下降段收油门，会把"加速型"油门响应每轮清零，短平飞段永远建不起速度、飞机寸步难行——可用的版本全程给固定前进油门，只切换升降舵。段长约 5s；循环到坪上方约 40ft 结束。参数表见 [USAGE.zh-CN.md](USAGE.zh-CN.md)。

## 7. 垂直降落 —— `_land.py`

升降舵控下沉率，目标下沉率按离地高度分级——高处快、贴地拉平：AGL >30ft 用 −6 ft/s，15–30 用 −3.5，6–15 用 −2，<6 用 −1。升降舵 = 前馈 `目标/11` + 对实测下沉率的小比例项；下沉超 12 ft/s 硬拉平。油门保持中性、原地下降。触地由 `SIM ON GROUND` / AGL 判定。

## 8. 场景与触发（`config.py` → `SCENARIOS`）

操作员 arm 一个诱因；`monitor_loop` 在飞机进入触发区**半径**内*且*在**高度带（MSL）**内时自动触发（需先在区外被看到一次）。停机坪 **ZKMZN**：23.14125, 113.27712, 168ft。

| 诱因 | 触发中心 | 半径 | 高度带 | 触发时效果 |
|---|---|---|---|---|
| 风切变 | 23.13909, 113.27622 | 50 m | 382–582 ft | 末端横向阵风：注入 `VELOCITY_BODY_X` 横移 + 一定 `VELOCITY_BODY_Y` 下沉（正弦）。断 AP 后飞机被动悬停，被试实时对抗阵风。HMI 接管请求 |
| 自驾故障 | 23.13957, 113.27645 | 50 m | 260–460 ft | `/ap_stop` + AUTO_STOP（交还被试）；HMI 接管请求。触发区已挪到与风切变共用的进近线上 |
| 障碍物 | 23.141696, 113.276120 | 60 m | 250–500 ft | 仅 HMI 提示；AP 继续飞坪。进近目标取**高度带中值（≈375ft）**，与楼顶齐平而非从上方飞过 |

风切变下沉只有在**密集、大值** `VELOCITY_BODY_Y` 注入（≈ −50 ft/s、高频）时才生效；小值/稀疏几乎无效。

## 9. 已知局限 / 下一步

- **横向转向。** 无航向输入时，从几何对齐的起点航迹保持得不错，但长距离可能偏。`VELOCITY_BODY_X` 修不了（写入不生效），强写航向又会震荡。干净的解法是用**方向舵**做交叉航迹闭环（真实的飞行员操纵）——`_rudder_test.py` 是它的探针。
- **传送后中性油门漂移**——已用"现读 + 绝对限幅"处理，但若要精确速度值得进一步理解。
